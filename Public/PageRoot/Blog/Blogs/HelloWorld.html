How is this website being ran?

At the time of writing this article there has been a prevevious prototype website already develped. In this previous version I used the web page render.com, this page allowes you to upload a github repositiory that contains node.js code to render a node-app online. The render page is a good way to test how the server in runs production before spending reasorces alocating a dedicated machine to host the page. The old page should not be accessible by the time you are reading this but here is a quick snapshot of the old page!

To keep track of version changes I use my personal github account to create a repository in which I am able to upload changes and updates for the pages. Using github has been a greate experiance more so given that I can directly push changes and simply request an update on the render dashboard.

I am using node.js to handle requests by the user. I thought that using something different like wordpress or a webpage editor like wix, it would be too easy and somewhat restrictive since i would not be able to have full control. coding something with multiple purpucess would be a nightmare to navigate and eddit code. I am not using any special frameworks like angular nor handlebars, the reasoning is that I what to see what i can get away with using the bare minimum components.

In summary: render.com + github.com + node.js + html + css = [insert name here]
